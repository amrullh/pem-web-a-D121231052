Bagaimana Internet Bekerja Secara Umum?

Secara fundamental, internet adalah "jaringan dari jaringan" (network of networks) global yang menghubungkan miliaran perangkat di seluruh dunia. Ini bukan satu entitas tunggal, melainkan jaringan desentralisasi yang terdiri dari jaringan akademik, bisnis, pemerintah, dan pribadi yang lebih kecil.

Komunikasi ini diatur oleh serangkaian aturan yang disebut protokol. Protokol inti yang menjadi tulang punggung internet adalah TCP/IP (Transmission Control Protocol / Internet Protocol).

IP (Internet Protocol): Bertugas seperti GPS atau sistem alamat pos. Setiap perangkat yang terhubung ke internet memiliki alamat IP unik. IP bertugas mengarahkan paket data dari sumber ke tujuan yang benar di seluruh jaringan.

TCP (Transmission Control Protocol): Bekerja di atas IP. Tugasnya adalah memastikan data terkirim dengan andal. Sebelum dikirim, data besar (seperti gambar atau video) dipecah menjadi bagian-bagian kecil yang disebut "paket" (packets). TCP memastikan semua paket tiba di tujuan, memeriksa apakah ada yang hilang atau rusak, dan menyusunnya kembali dalam urutan yang benar.

Secara fisik, data ini berjalan melalui infrastruktur masif: kabel serat optik di bawah laut, satelit di luar angkasa, kabel tembaga, menara seluler, dan router di rumah atau kantor Anda. ISP (Internet Service Provider) Anda adalah gerbang yang menghubungkan Anda ke jaringan global yang lebih luas ini.

Definisi dan Cara Kerja HTTP & HTTPS

HTTP (Hypertext Transfer Protocol) adalah protokol lapisan aplikasi yang menjadi fondasi komunikasi data di World Wide Web. Anggap saja ini sebagai "bahasa" atau "format pesan" yang disepakati oleh browser (klien) dan server web.

Cara kerjanya berdasarkan model request-response (permintaan-tanggapan):

  1. Request (Permintaan): Browser Anda mengirim pesan HTTP (HTTP request) ke server untuk meminta sumber daya. Contohnya, "GET /halaman/kontak.html" yang berarti "Tolong berikan saya file kontak.html".
  2. Response (Tanggapan): Server memproses permintaan itu dan mengirimkan kembali pesan HTTP (HTTP response) yang berisi sumber daya yang diminta (misalnya, kode HTML halaman) atau kode status (seperti "404 Not Found" jika file tidak ada).

HTTPS (Hypertext Transfer Protocol Secure) adalah evolusi dari HTTP. Perbedaan kuncinya adalah keamanan. HTTPS menggunakan protokol tambahan yang disebut SSL/TLS (Secure Sockets Layer/Transport Layer Security) untuk mengenkripsi semua komunikasi antara browser Anda dan server.

Enkripsi ini mengacak data, sehingga jika ada peretas yang mencoba "menguping" koneksi (misalnya di Wi-Fi publik), mereka hanya akan melihat data acak yang tidak dapat dibaca. HTTPS juga memverifikasi identitas server melalui Sertifikat SSL/TLS, yang diwakili oleh ikon gembok di browser Anda. Ini memastikan Anda terhubung ke server yang benar (misalnya, bank Anda) dan bukan ke situs penipu.

Definisi dan Cara Kerja Nama Domain

Nama Domain (Domain Name) adalah alamat yang mudah diingat yang digunakan manusia untuk mengidentifikasi lokasi di internet, seperti `google.com` atau `detik.com`. Ini adalah pengganti yang ramah pengguna untuk Alamat IP (IP Address).

Komputer dan server di internet tidak saling menemukan menggunakan nama; mereka menggunakan string numerik yang disebut Alamat IP (misalnya, `142.250.190.78`). Akan sangat sulit bagi manusia untuk mengingat ribuan alamat IP. Nama domain memecahkan masalah ini. Ini berfungsi persis seperti "nama kontak" di ponsel Anda. Anda tidak perlu menghafal nomor telepon, Anda cukup menyimpan "Ibu" atau "Budi", dan ponsel Anda yang akan menerjemahkan nama itu ke nomor yang benar saat Anda menelepon.

Nama domain memiliki struktur hierarkis. Misalnya, pada `www.google.com`:

.com: Ini adalah Top-Level Domain (TLD). Ada banyak TLD lain seperti .org, .net, .id (khusus negara), atau .tech.

google: Ini adalah Second-Level Domain (SLD). Ini adalah bagian unik yang biasanya Anda beli atau daftarkan.

www: Ini adalah subdomain.

Organisasi global bernama ICANN (Internet Corporation for Assigned Names and Numbers) mengoordinasikan sistem nama domain ini untuk memastikan setiap alamat unik dan terpetakan dengan benar.

Sumber Referensi: ICANN (Internet Corporation for Assigned Names and Numbers). (2023). "What's a Domain Name?". & Cloudflare. (2023). "What is a Domain Name?".

Definisi dan Cara Kerja Hosting

Hosting (Web Hosting) adalah layanan yang menyediakan ruang penyimpanan (storage) dan sumber daya komputasi di server agar website Anda dapat diakses oleh siapa saja di internet, kapan saja.

Jika nama domain adalah "alamat" unik rumah Anda, maka hosting adalah "tanah dan bangunan" tempat Anda menyimpan semua perabotan. "Perabotan" ini adalah file-file website Anda: file `index.html`, `style.css`, file JavaScript, gambar, video, dan database.

Server hosting pada dasarnya adalah komputer berkinerja tinggi yang selalu menyala (24/7) dan terhubung ke internet dengan koneksi berkecepatan sangat tinggi. Ketika seseorang mengetik nama domain Anda di browser, browser tersebut akan diarahkan ke server hosting ini. Server kemudian akan "menyajikan" atau mengirimkan file-file website Anda ke browser pengunjung, yang kemudian menampilkannya sebagai halaman web.

Ada berbagai jenis hosting, mulai dari Shared Hosting (paling murah, seperti berbagi apartemen dengan banyak orang), VPS (Virtual Private Server) (lebih terkontrol, seperti unit kondominium), hingga Dedicated Server (paling mahal, seperti memiliki seluruh gedung sendiri).

Definisi dan Cara Kerja DNS

DNS (Domain Name System) adalah sistem terdistribusi global yang berfungsi sebagai "buku telepon internet". Peran utamanya adalah menerjemahkan Nama Domain yang mudah dibaca manusia (seperti `google.com`) menjadi Alamat IP yang dimengerti mesin (seperti `142.250.190.78`).

Proses ini disebut "Resolusi DNS" dan terjadi sangat cepat di latar belakang setiap kali Anda mengunjungi situs web. Berikut adalah langkah-langkah yang disederhanakan:

  1. Permintaan Awal: Anda mengetik `google.com` di browser. Browser Anda akan bertanya ke DNS Resolver (biasanya disediakan oleh ISP Anda), "Apa IP untuk `google.com`?"
  2. Query ke Root Server: Jika Resolver tidak tahu (tidak ada di cache), ia akan bertanya ke Root Server (server puncak). Root Server akan berkata, "Saya tidak tahu, tapi saya tahu siapa yang mengurus `.com`. Tanyakan pada mereka."
  3. Query ke TLD Server: Resolver kemudian bertanya ke TLD Server (server untuk `.com`). TLD Server menjawab, "Saya tidak tahu IP pastinya, tapi saya tahu server mana yang bertanggung jawab atas domain `google.com`. Tanyakan pada Authoritative Nameserver mereka."
  4. Jawaban dari Authoritative Server: Akhirnya, Resolver bertanya ke Authoritative Nameserver (yang dikelola oleh Google). Server ini memegang catatan resmi dan menjawab, "Ya, IP untuk `google.com` adalah `142.250.190.78`."
  5. Resolusi: Resolver memberikan jawaban IP ini kembali ke browser Anda. Agar efisien, Resolver juga menyimpan jawaban ini di cache untuk sementara, sehingga jika Anda bertanya lagi nanti, prosesnya bisa dilewati.

Definisi dan Cara Kerja Browser

Browser (Web Browser) adalah aplikasi perangkat lunak yang diinstal di perangkat Anda (seperti Chrome, Firefox, Safari) yang memiliki satu tugas utama: mengambil sumber daya dari web dan menampilkannya kepada pengguna.

Browser bukanlah internet itu sendiri; ia adalah alat untuk mengakses dan "menerjemahkan" internet. Saat Anda memasukkan URL, browser melakukan serangkaian langkah yang kompleks yang disebut Critical Rendering Path (Jalur Perenderan Kritis):

  1. Mengambil (Fetch): Menggunakan DNS untuk menemukan IP server, lalu mengirimkan permintaan HTTP/HTTPS ke server tersebut untuk meminta file (terutama file HTML).
  2. Mem-parsing (Parse): Setelah file HTML diterima, mesin perender (rendering engine) browser (seperti Blink di Chrome) akan memindai dan mem-parsing kode HTML untuk membangun struktur yang disebut DOM (Document Object Model) Tree. Ini adalah representasi internal halaman dalam bentuk pohon objek.
  3. Membangun CSSOM: Browser juga akan menemukan link ke file CSS. Ia akan mengambil dan mem-parsing file CSS untuk membangun CSSOM (CSS Object Model) Tree, yang berisi semua informasi gaya (style) untuk setiap elemen.
  4. Render Tree: Browser kemudian menggabungkan DOM dan CSSOM untuk membuat Render Tree. Pohon ini hanya berisi elemen-elemen yang akan ditampilkan secara visual di layar.
  5. Tata Letak (Layout): Browser menghitung dengan tepat di mana setiap elemen harus ditempatkan di layar (koordinat X/Y) dan berapa ukurannya (lebar/tinggi).
  6. Melukis (Paint): Akhirnya, browser "melukis" atau merasterisasi piksel-piksel ke layar, mengubah elemen-elemen abstrak dari Render Tree menjadi gambar visual yang Anda lihat.

Selain itu, browser juga memiliki JavaScript Engine (seperti V8 di Chrome) untuk menjalankan kode JavaScript, yang memungkinkan halaman web menjadi interaktif (seperti slider, pop-up, atau game).
