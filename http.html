Definisi dan Cara Kerja HTTP & HTTPS

HTTP (Hypertext Transfer Protocol) adalah protokol lapisan aplikasi yang menjadi fondasi komunikasi data di World Wide Web. Anggap saja ini sebagai "bahasa" atau "format pesan" yang disepakati oleh browser (klien) dan server web.

Cara kerjanya berdasarkan model request-response (permintaan-tanggapan):

  1. Request (Permintaan): Browser Anda mengirim pesan HTTP (HTTP request) ke server untuk meminta sumber daya. Contohnya, "GET /halaman/kontak.html" yang berarti "Tolong berikan saya file kontak.html".
  2. Response (Tanggapan): Server memproses permintaan itu dan mengirimkan kembali pesan HTTP (HTTP response) yang berisi sumber daya yang diminta (misalnya, kode HTML halaman) atau kode status (seperti "404 Not Found" jika file tidak ada).

HTTPS (Hypertext Transfer Protocol Secure) adalah evolusi dari HTTP. Perbedaan kuncinya adalah keamanan. HTTPS menggunakan protokol tambahan yang disebut SSL/TLS (Secure Sockets Layer/Transport Layer Security) untuk mengenkripsi semua komunikasi antara browser Anda dan server.

Enkripsi ini mengacak data, sehingga jika ada peretas yang mencoba "menguping" koneksi (misalnya di Wi-Fi publik), mereka hanya akan melihat data acak yang tidak dapat dibaca. HTTPS juga memverifikasi identitas server melalui Sertifikat SSL/TLS, yang diwakili oleh ikon gembok di browser Anda. Ini memastikan Anda terhubung ke server yang benar (misalnya, bank Anda) dan bukan ke situs penipu.
