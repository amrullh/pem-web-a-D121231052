🗺️ DNS (Domain Name System)

Buku Telepon Raksasa Milik Internet.

Menerjemahkan Nama Menjadi Angka

DNS (Domain Name System) adalah sistem terdistribusi global yang berfungsi sebagai "buku telepon internet". 📖 Tadi kita tahu bahwa domain adalah "nama kontak" dan server punya "nomor telepon" (Alamat IP).

Nah, DNS adalah layanan yang mencari tahu "nomor" siapa saat Anda mengetik "nama".

Proses ini disebut "Resolusi DNS". Ini terjadi sangat cepat di latar belakang. Begini alurnya:

  1. Permintaan Awal: Anda mengetik `google.com` di browser. Browser Anda bertanya ke DNS Resolver (biasanya dari ISP Anda), "Bro, IP-nya `google.com` berapa?"
  2. Query ke Root Server 🌍: Resolver (jika tidak tahu) bertanya ke Root Server. Root Server tidak tahu, tapi dia bilang, "Coba tanya server `.com`."
  3. Query ke TLD Server (.com) 🏷️: Resolver bertanya ke TLD Server (server `.com`). Server ini juga tidak tahu, tapi dia bilang, "Oh, `google.com`? Tuh tanya ke servernya Google langsung (Authoritative Nameserver)."
  4. Jawaban dari Authoritative Server 🎯: Resolver bertanya ke Authoritative Nameserver Google. Server ini memegang catatan resmi dan menjawab, "Yoi, IP-nya `172.217.14.228`."
  5. Resolusi: Resolver memberikan IP itu ke browser Anda. Browser Anda sekarang tahu "nomor telepon" server Google dan bisa menghubunginya.

Untungnya, ada Caching! Resolver Anda akan "mengingat" jawaban itu. Jadi, jika teman Anda juga membuka `google.com`, proses 1-5 ini tidak perlu diulang. Resolver langsung memberikan jawaban yang sudah disimpan. Cepat!
